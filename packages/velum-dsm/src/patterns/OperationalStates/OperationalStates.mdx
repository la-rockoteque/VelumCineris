# Operational States

These are the standard behavior patterns for async and high-friction flows in Velum DSM.

## Shared Contract

- Keep layout stable while state changes. Replace content, not the surrounding structure.
- Show status near the affected region. Do not force users to scan the whole page for feedback.
- Pair disabled controls with reason text when the reason is not obvious from context.
- Keep recovery actions adjacent to the state they resolve: retry near errors, save near edited content, pagination near the dataset.
- Use contained messaging for local feedback instead of global noise.

## Pattern Set

- `Patterns/Loading`
- `Patterns/Scrolling`
- `Patterns/Pagination`
- `Patterns/Saving`
- `Patterns/Disabled`
- `Patterns/Error Management`

Use these as the baseline interaction contract across apps and tools.
