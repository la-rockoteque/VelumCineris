# Error Management

Errors should be specific, local to the failing surface, and recoverable.

## Contract

- Place the error message inside the affected workspace or section.
- Explain the failure in concrete terms and suggest the next action.
- Keep retry and recovery actions adjacent to the error.

## Recommended Use Cases

- Failed API reads or writes
- Missing validation sheets
- Integration failures with partial success or retry semantics
