# Disabled

Disabled controls need surrounding context so the user understands what prerequisite is missing.

## Contract

- Pair disabled controls with explicit reason text when the requirement is not obvious.
- Keep the explanation adjacent to the blocked action.
- Reserve disabled state for unavailable actions, not passive information.

## Recommended Use Cases

- Actions that require row selection
- Submit buttons gated on validation
- Integrations blocked by missing credentials or settings
